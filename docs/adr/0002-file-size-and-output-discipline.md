# ADR 0002: 文件行数限制与输出纪律

Date: 2026-06-14  
Status: Accepted

## Context

重构过程中发现两个反复出现的问题：
1. 单个文件行数无约束，导致模块膨胀（UIController 已超过 5000 行）
2. 每次回复连续执行多个工具调用，用户无法在中间步骤介入或验证

## Decision

**每个代码文件不超过 500 行。** 超出必须拆分为多个文件。

**每次回复只执行一个工具调用**，执行后停止，等待用户确认再继续。

## Consequences

- 模块天然保持小而聚焦，超出 500 行是职责过重的信号
- 用户在每一步都有机会介入，避免大批量错误积累
- 新建文件前必须先估算行数
